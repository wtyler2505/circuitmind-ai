# Definition of Done

## Code Quality
- [ ] TypeScript types are exhaustive (no `any` without justification).
- [ ] ESLint passes with zero warnings.
- [ ] Unit tests cover critical logic (especially in services).
- [ ] Logic is optimized for performance (no unnecessary re-renders in Contexts).

## Visual & UX
- [ ] UI adheres to the "Neon-Cyber" style guide.
- [ ] Panning and zooming on the canvas are smooth.
- [ ] Component states (hover, select, active) are visually clear.
- [ ] Dark mode/Light mode contrast is accessible.

## Documentation
- [ ] Feature spec and tech design are up to date.
- [ ] `docs/03-logs/implementation-log.md` updated with the change.
- [ ] New components or services added to the relevant context file.

## AI & Data
- [ ] `AIContext` is updated if new metadata is available.
- [ ] State snapshots (Git) capture the new features correctly.
